Business Problem

Placement cells manage thousands of student–company interactions using manual Excel sheets — no pipeline visibility, no prediction, no benchmarking. This project replaces manual processes with a complete analytics system.

What I Built

  • 4-table Star Schema SQL database (students, companies, applications, interviews) with 46K+ records, PKs, FKs, and performance indexes
  • 35 SQL analytics queries including CTEs, window functions (RANK, LAG, NTILE), and subqueries for 8 business KPIs
  • Python EDA with 14 engineered features including a composite placement score; Random Forest model achieving 86.1% accuracy and AUC 0.73
  • 4-page Power BI dashboard with 10 DAX measures, drill-through pages, and dynamic slicers — reduced manual reporting by 30%
  • Identified internship count as the #1 placement predictor (feature importance: 0.21), outranking CGPA

Social Media Sentiment & Brand Intelligence Dashboard

PROJECT 02 · NLP & BRAND ANALYTICS
30K+Posts Analysed
82%NLP Accuracy
3Brands Tracked

Business Problem

Brands receive thousands of daily social mentions with no systematic way to detect sentiment shifts before they become PR crises. This project builds a real-time brand health monitoring system.

What I Built

  • 7-step NLP preprocessing pipeline: URL/mention removal, stop word filtering, lemmatization using NLTK — cleaning 30,000 posts
  • Dual sentiment scoring with VADER (lexicon-based) and TextBlob (pattern-based); ensemble model combining both scores achieved 82% accuracy
  • LDA topic modelling with 5 components per brand — surfaced camera quality and after-sales service as top cross-brand complaint themes
  • Z-score spike detection (threshold: |z| > 2) flagging abnormal negative sentiment days as crisis alerts
  • Power BI dashboard with share-of-voice trend, engagement-weighted sentiment, and 3-brand competitor comparison

Redrob Candidate Ranking Engine

PROJECT 04 · RANKING & MLOps
100K+Profiles Scored
Top 100Leaderboard
3-TierFraud Filter

Business Problem

Recruiting top-tier AI Engineers is plagued by resume padding and volume overload. This engine efficiently filters 100,000+ synthetic profiles to identify authentic talent while penalizing fabricated "honeypot" resumes.

What I Built

  • Streamlined ingestion and processing pipeline for massive datasets using min-heap selectors to maintain a top-100 leaderboard without expensive global sorts.
  • Feature engineering module using optimized regex to extract core AI signals like FAISS, Vector DBs, XGBoost, LightGBM, and MLOps.
  • 3-Tier Honeypot Detection system applying multiplicative penalties (0.05x to 0.6x) for mathematical impossibilities, AI buzzword stuffing, and excessive skill breadth.
  • Natural language reasoning generator explaining why each of the final 100 candidates was selected into the final CSV.
